Evaluation 

 

We have created a Todo app which stores Todo items as a list. There is functionality to add, remove, and edit items, and the data is stored inside of a SQL Server database. 

The collaboration in the group was fluent, and everyone contributed to the process of making the app. We did have some conflicts and sometimes different opinions about the functionality of the app etc., and we did overcome these conflicts and could manage a good work together. 

We created a development branch which was meant to be a branch that we branch of from for different features, but as the time pressure increased during the work, we branched of only once from development and at the end we merged development to the master branch. 

The collaboration and management of the git repo worked well. We also managed to work according to the mob programming method. All of us wrote code every day and during this time, the other two members of the group were trying to help, navigate, give ideas and tips and contribute to a good quality code and design. 

What we could have done better was to manage the git workflow better and branch off from our development branch for every feature separately so that the work could get better organized and avoid merge conflicts.
